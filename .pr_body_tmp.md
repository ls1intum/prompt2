## ✨ What is the change?

Update the prompt-sdk dependency across all backend microservices from the old module path `github.com/ls1intum/prompt-sdk` to the new canonical path `github.com/prompt-edu/prompt-sdk`, and bump the version to v1.1.0. All import paths in Go source files and all `go.mod` / `go.sum` files have been updated accordingly.

## 📌 Reason for the change / Link to issue

The prompt-sdk module was moved to the `prompt-edu` GitHub organisation. Keeping the old `ls1intum` import path would cause build failures as the old path is no longer the source of truth. This change ensures all seven servers (assessment, core, interview, intro_course, self_team_allocation, team_allocation, template_server) resolve the SDK from the correct registry path and pick up the latest v1.1.0 release.

## 🧪 How to Test

1. Run `make server` (or `go build ./...` inside any of the seven server directories) and confirm there are no import errors.
2. Run `go test ./...` inside each server to verify the test suite still passes.
3. Optionally start the full stack with `make db && make server` and confirm the services start without panics.
